# Git Workflow - Інструкція

## Налаштування завершено ✓

Ваш Git репозиторій налаштований для роботи з локальними змінами та завантаженням на сервер.

## Основні команди

### Перевірка статусу
```bash
git status
# або коротко:
git st
```

### Додавання файлів до staging area
```bash
# Додати всі змінені файли
git add .

# Додати конкретний файл
git add шлях/до/файлу.php
```

### Створення коміту
```bash
git commit -m "Опис змін"
# або використовуючи alias:
git cm "Опис змін"
```

### Перегляд історії
```bash
git log --oneline
# або красивий вигляд:
git lg
```

## Робочий процес (Workflow)

### 1. Робота локально
```bash
# 1. Перевірте статус
git st

# 2. Внесіть зміни у файли

# 3. Додайте зміни до staging
git add .

# 4. Створіть коміт
git cm "Опис ваших змін"
```

### 2. Завантаження на сервер (з підтвердженням)

#### Варіант А: Використання скрипта (рекомендовано)
```powershell
# Запустіть скрипт з описом змін
.\deploy.ps1 "Опис змін"

# Або без опису (використає стандартне повідомлення)
.\deploy.ps1
```

Скрипт автоматично:
- Перевірить статус
- Покаже незакомічені зміни
- Отримає останні зміни з сервера
- Запитає підтвердження перед завантаженням
- Завантажить зміни на сервер

#### Варіант Б: Ручне завантаження
```bash
# 1. Отримати останні зміни з сервера
git fetch origin

# 2. Перевірити відмінності
git log HEAD..origin/main --oneline

# 3. Якщо потрібно, отримати зміни
git pull origin main

# 4. Завантажити свої зміни
git push origin main
```

## Корисні aliases (скорочення)

- `git st` → `git status`
- `git co` → `git checkout`
- `git br` → `git branch`
- `git cm` → `git commit -m`
- `git lg` → красивий вигляд історії

## Важливо!

1. **Завжди перевіряйте зміни** перед комітом: `git diff`
2. **Пишіть зрозумілі повідомлення** в комітах
3. **Не комітьте конфіденційні дані** (config.php вже в .gitignore)
4. **Використовуйте скрипт deploy.ps1** для безпечного завантаження

## Перший коміт

Якщо ви ще не зробили перший коміт:

```bash
# 1. Додати всі файли
git add .

# 2. Створити перший коміт
git commit -m "Початковий коміт проекту"

# 3. Завантажити на сервер
git push -u origin main
```

## Відміна змін

```bash
# Відмінити зміни в файлі (незакомічені)
git checkout -- файл.php

# Видалити файл з staging area
git reset HEAD файл.php

# Відмінити останній коміт (залишити зміни)
git reset --soft HEAD~1
```
