# Git Workflow - Інструкція з роботи

## Налаштування

Репозиторій налаштований для роботи з GitHub:
- **Remote**: `origin` → `https://github.com/valet343/purelife.git`
- **Гілка**: `main`

## Робочий процес

### 1. Локальна робота

Робіть зміни локально у файлах проекту. Git відстежує всі зміни.

### 2. Перевірка змін

```powershell
git status
```

### 3. Додавання змін до коміту

```powershell
# Додати всі зміни
git add .

# Або додати конкретні файли
git add path/to/file.php
```

### 4. Створення коміту

```powershell
git commit -m "Опис ваших змін"
```

### 5. Відправка на сервер (після підтвердження)

```powershell
git push origin main
```

## Автоматичний скрипт деплою

Для зручності використовуйте скрипт `deploy.ps1`:

```powershell
# З повідомленням за замовчуванням
.\deploy.ps1

# З власним повідомленням
.\deploy.ps1 "Виправлено баг в каталозі"
```

Скрипт автоматично:
1. Перевіряє статус репозиторію
2. Пропонує додати зміни до коміту
3. Створює коміт
4. Запитує підтвердження перед відправкою на сервер
5. Відправляє зміни на GitHub

## Корисні команди

### Перегляд історії
```powershell
git log --oneline -10
```

### Перегляд змін
```powershell
git diff
git diff --staged
```

### Скасування змін
```powershell
# Скасувати зміни в файлі (не закомічені)
git checkout -- path/to/file.php

# Прибрати файл з staging area
git reset HEAD path/to/file.php
```

### Отримання змін з сервера
```powershell
git pull origin main
```

## Важливо

- **config.php** та **admin/config.php** виключені з Git (містять конфіденційні дані)
- Логи та кеш також виключені
- Завжди перевіряйте зміни перед комітом
- Використовуйте зрозумілі повідомлення комітів
